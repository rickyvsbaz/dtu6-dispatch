Complete Slack App Setup Guide
Step 1 — Create a Slack App
Go to api.slack.com/apps
  1. Sign in at api.slack.com/apps with your Slack work account
  2. Click Create New App → From scratch
  3. Name it DTU6 Dispatch Bot, pick your workspace → Create App
Step 2 — Enable Incoming Webhooks
This is how the bot posts messages
  1. In the left sidebar of your app settings, click Incoming Webhooks
  2. Toggle Activate Incoming Webhooks to On
  3. Scroll down and click Add New Webhook to Workspace
  4. Choose your dispatch channel (e.g. #dispatch or #az-daily-operations) → click Allow
  5. You'll see a Webhook URL that starts with https://hooks.slack.com/services/
  6. Click Copy
📋 The webhook URL looks like: https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXXXXXX
Step 3 — Paste Webhook into the Bot
Left sidebar → Slack Config
  1. Paste the webhook URL into the Slack Config field on the left sidebar
  2. Click the ✓ button — you'll get a test message in your channel confirming it works
  3. The webhook URL is automatically saved in your browser so you only enter it once
⚠️ The webhook URL is sensitive — anyone with it can post to your channel. Keep it private and only paste it into this tool.
Step 4 — For Each Dispatcher
Share the webhook, not the page
Each dispatcher opens your GitHub Pages URL in their browser, then pastes the webhook URL once. It saves automatically after that. You can share the same webhook across all dispatchers — it posts to the same channel regardless of who triggers it.

Want the webhook pre-loaded so dispatchers never have to enter it? Just ask and I can bake it directly into the HTML file — then it works out of the box for everyone.
Bot Token — for Auto-Delete & Direct Posting
Optional but recommended
The Bot Token (xoxb-) enables auto-delete at 10 PM MST and direct channel posting.
  1. In your Slack App settings → OAuth & Permissions → add these scopes:
    chat:write   channels:history   groups:history
  2. Reinstall the app to your workspace after adding scopes
  3. Copy the Bot User OAuth Token and paste it in the Bot Token field in Slack Config
ℹ️ chat:delete is only available on paid Slack plans. The auto-delete uses a timed browser call instead — it works as long as the dispatch tab stays open until 10 PM. See below for a server-based alternative.
Auto-Delete Without Keeping Browser Open
Free server-based solution using cron-job.org
  1. Go to cron-job.org and create a free account
  2. After posting today's message, copy the message timestamp (ts) from the browser console — it looks like 1741234567.123456
  3. Create a cron job set to run at 05:00 UTC daily (= 10 PM MST)
  4. Set it to call: POST https://slack.com/api/chat.delete
  5. Headers: Authorization: Bearer xoxb-your-token
  6. Body: {"channel":"#dispatch","ts":"PASTE_TS_HERE"}
💡 Easier alternative: just have dispatch manually delete the message at end of day, or set a Slack channel reminder to do it.
